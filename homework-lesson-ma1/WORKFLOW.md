Agent1: Customer definer
1. Визначити з електронного листа електронну адресу відправника
2. Знайти, чи е така адреса у CRM. якщо є, визначити відправника та компанію (get_contact).
3. Визначити домен та знайти компанію по домену (get_company).
4. Якщо компанії нема, знайти в інтернеті по домену компанію, визначити її характеристики через (check_opendatabot).
5. Створити карточку компанії, створити карточку співробітника замовника (create_company, create_contact).
6. Перевірити компанію по правдам перевірки юриста.
7. Визначити електронну адресу отримувачів. Якщо адреса отримувача з домену techsvit.ua, знайти його ID у CRM (get_user:
   GET https://YOUR_BITRIX_DOMAIN/rest/1/WEBHOOK_CODE/user.get.json?FILTER[EMAIL]=user@example.com)
8. Визначити серед користувачів користувача з роллю КАМ та призначити його відповідальним.

4. Передати лист агенту, який визначає позиції замовлення.

Agent2: Order definer
1. Визначити перелік товарів, яки запитуються
2. По кожній позиції знайти в інтернеті пряму відповідність SKU на сайтах ERC, Asbis, MTI та ще трьох інших українських сайтів з посиланнями.
   Якщо прямої відповідності не знайдено, знайти до 3-х найближчих аналогів.
5. Для аналогів зробити порівняльну таблицю і зробити коротку порівняльну таблицю.
5. По кожному знайденому SKU зробити запит ціни у вендорів
3. Записати запит у RAG базу з метаданими ID компанії, відправника, та відповідального.
